# FraudShield — Deploy Guide

## Run Locally

```bash
pip install -r requirements.txt
python app.py
# Open http://localhost:5000
```

The app works immediately in **demo mode** (rule-based heuristics).
To use the trained XGBoost model, run the training step first:

```bash
# Place your dataset CSV in this folder, then:
python train_model.py
python app.py
```

## Deploy to Vercel

### Option A — Vercel CLI (recommended)
```bash
npm install -g vercel
vercel login
vercel --prod
```

### Option B — Vercel Dashboard
1. Go to https://vercel.com/new
2. Click **Import Git Repository**
3. Select your GitHub repo
4. Click **Deploy** — Vercel auto-detects the Python app via `vercel.json`

### Note on the model file
`model/fraud_model.pkl` is excluded from git (`.gitignore`).
Vercel runs in **demo mode** by default.

To include the trained model:
1. Remove `model/fraud_model.pkl` from `.gitignore`
2. Train the model locally: `python train_model.py`
3. Commit and push the pkl file
4. Redeploy

> **Keep in mind:** Vercel has a 50MB file size limit per deployment.
> XGBoost models are typically 2–10MB, well within limits.
